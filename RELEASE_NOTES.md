# OTelVerse v1.0.0 Stable Release

We are thrilled to announce the **v1.0.0 Stable Release** of the OTelVerse Platform! 🎉 
This milestone marks the culmination of weeks of active development to create the ultimate unified observability experience built purely on OpenTelemetry.

## What's Included

* **10 Core Products**: 
  * Unified Platform (Go & ClickHouse)
  * Pipeline Builder (React Flow visual editor)
  * Frontend Wrapper (Browser auto-instrumentation)
  * Pipeline Optimizer (AI-driven heuristic engine)
  * Chaos Engine (OTLP-native failure injection)
  * Session Replay (DOM snapshot recording)
  * Edge Agent (Rust-based local collector)
  * Robotics SDK (ROS 2 & Gazebo tracing)
  * Dev Desktop (Cross-platform GUI)
  * Integration Kits (Docker Compose & Helm)
* **Alerting & Metrics**: Full-stack alerting engine with multi-channel support and dynamic metrics dashboards.
* **UQL Enhancements**: Advanced OpenTelemetry querying with our Unified Query Language (UQL), featuring cross-signal joins and distributed aggregations.
* **E2E Validation**: Comprehensive automated testing and CI pipelines validating the entire platform.
* **Hardening & Security**: OAuth2 proxy authentication, PostgreSQL persistence with in-memory fallbacks, ClickHouse pooling, and automated security scanning.
* **Governance**: The project is now officially licensed under Apache 2.0. We've established a Code of Conduct and clear contributor guidelines to welcome the community.

## Getting Started

Check out the [Quick Start Guide](https://docs.otelverse.io) to deploy the platform locally using Docker Compose, or grab the latest Helm charts for Kubernetes.

Thank you to all the contributors and community members who helped make this release possible!
