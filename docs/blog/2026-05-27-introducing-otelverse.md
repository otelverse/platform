We're excited to announce **OTelVerse**, the complete OpenTelemetry ecosystem for developers!

OpenTelemetry is the future of observability, but setting up the pipeline, configuring visual dashboards, injecting faults, and understanding trace data can be incredibly complex. OTelVerse unifies all these aspects into a single, cohesive platform.

### What's in OTelVerse?
- **Unified Platform**: A central control plane to view traces, session replays, and manage pipelines.
- **Visual Pipeline Builder**: Drag-and-drop ReactFlow interface to configure your OTel Collector processors, receivers, and exporters.
- **Chaos Engine**: An OTLP-native proxy that injects latency and errors right into your trace stream to test system resilience.
- **AI Optimizer**: Automatically detects PII and high-error traces, recommending robust sampling policies to save costs.
- **Session Replay**: Links DOM snapshots directly to traces, so you see exactly what the user saw when the error occurred.
